Html entities

There are three reserved characters in HTML: < > and &. You should always use HTML entities for these three characters. HTML entities can't be represented by plain text.

Quotes

The basic quotes are straight lines that don't hug the text: ' and ". If you want the ‘singles’ and “doubles” to hug, then you need to use entities.

  • &ldquo; + &rdquo; for doubles
  • &lsquo; + &rsquo; for doubles

Page specific styling

<style> 
  body { 
    color: #0000FF;    /* Blue */ 
  }  
</style>

Padding vs border vs margin

The content is defined in the html. The padding surround this and takes the same background colour as the content. You can define this in a variety of units and methods (the same methods apply to margin and border):

  • You can define the padding for all sides with one value: padding: 2em;
  • You can define the vertical padding (same for the top and bottom) and the horizontal padding (same for each side): padding: 20px 30px;
  • You can define clockwise, starting at the top: padding: 20px 0 30px 35px; /* The top is 20px; the right has no padding, etc. */
  • You can define individually with padding-bottom: 10px;

For the border, you add a line around the padding. If you use the dashed, you can see that behind is the background-color still. The format is border: thickness style colour.

Margins surround the border and work like the margin but with some caveats: they don't take the background-color; they collapse vertically; and they are not part of the click area

Inline vs block

Inline boxes don't affect vertical spacing and width is determined by its content. The block elements can influence layout, their height is defined by their content, the width is set by the parent and block content always appear below the previous block element.

Preventing margin collapse

Pop an invisible element between them like so:

<div style='padding-top=1px;'><div>

If you don't mind the background-color changing then you could use padding as that never collapses.

Alignment

Text alignment differs to block alignment. The former can be set with text-alignment, whilst the latter can be set using auto-centering, floats or flexboxes.

To auto center, you use a horizontal margin of auto and this will center the block element. This requires that a width is set, otherwise the content will fill the whole width and the margin will be zero, on both sides.

  • Inline elements ignore vertical margins (eg. padding: 40px; would only apply 40px either side). Padding will display vertically but won't affect the layout.
  • If you have two adjecent block elements, one with bottom and the other with top margins, only the biggest margin between them is used. This is known as vertical margin collapse.
  • The only real difference between a <div> and a <span> is that the former is for block-level content while the latter is meant for inline content.
